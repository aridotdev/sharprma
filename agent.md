# WORKFLOW GUIDANCE

1. Gunakan .project untuk referensi utama
2. Think through the problem & read relevant files first
3. Create task list & get approval before major changes
4. Give high-level explanations of changes
5. Keep changes simple & minimal
6. Maintain architecture documentation
7. Never speculate - always read files first, use official docs as reference
8. State assumptions explicitly
9. When building: new branch → confirm → checkpoint → summary .md in /.doc
